Main core sub-systems

Game objects

Base of all entities in the scene. Can have the following components

  • Transform
    Alters the position, rotation and scale
  • Mesh
    3D mesh to be printed. Includes the amount of vertices and faces and has the option to see its normals
  • Texture
    2D image assigned to the mesh, can be changed to a checkboard
  • Camera
    Includes FOV, zNear, zFar and aspect ratio

Scene hierarchy

    Delete, reparent, create empty and create children

Mesh loading optimization

    With our own file format created for each mesh when added to the assets, loading of the meshes is quicker and lighter

UI docking

    Since the UI was made with ImGui, all window placements can be edited as user prefers

Particle system

    Detailed explanation of last assignment sub-system, containing two animated gif that shows editor and in-game functionality.

    In this Particle System there are 4 main elements:

  • Particle System: A component of GameObjects that holds a vector of Emmiters, that manages when the Emmiters start.
  • Emmiter: A class that holds a few basic parameters, like spawnRate, duration and maximum number of Particles, a series of Emmiter Modules, that set how the Particles will behave, and a Particle Object Pool.
  • Emmiter Module: An Emmiter Module is a class that affects 1 of the 4 stages of the Emmiter:
    • Spawn: The way the Emmiter will Spawn the Particles (In a constant rate, with a single burst, etc.).
    • Initialize: The values the Particle will have at the start of their individual life.
    • Update: The change of values of each Particle's life.
    • Render: The way each particle will be rendered.
  • Particle: A simple class that only stores its individual transformation and color.

    • This Particle Systems works in a tree structure way, where ParticleSystems hold Emmiters, that each holds a Particle Object Pool and a set of Modules.
